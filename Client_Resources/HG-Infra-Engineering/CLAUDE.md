## Requirements (Extracted from Call)

### What They Currently Use
- **MS Project (Enterprise version)** — for project planning
- **Manual processes** — for estimation, BOQ, design-to-execution workflow
- Multiple people working on planning and estimation simultaneously
- Large line items in BOQs (highway-scale projects)

### What They Need

#### 1. Project Planning System (HIGH priority)
- Replace or supplement MS Project
- Planning for highway/road construction projects
- Track: which engineer is where, how much work done, on-time or delayed
- Need visibility across multiple project sites
- 300+ users eventually (site supervisors, engineers, managers)

#### 2. Design → Estimation Pipeline (HIGH priority)
- Upload design documents
- System should auto-generate estimation from design
- Currently very manual, time-consuming, error-prone
- Many line items per project (highway-scale BOQs)
- Want AI-assisted estimation to reduce time and errors

#### 3. Procurement Management
- Track materials, purchase orders
- Link procurement to project planning
- Onsite's existing procurement module discussed as fit

#### 4. Execution Tracking
- Site-level daily progress tracking
- Which engineer at which site, work progress, timeline adherence
- DPR (Daily Progress Reports) automation

#### 5. Government Project Compliance
- NHAI, Railways, government project documentation
- Timeline and milestone tracking for government audits

### Scale & Complexity
- **300+ users** across sites (will need phased rollout)
- **Pan-India projects** — sites in multiple states simultaneously
- **Railway + Road + Metro** — multiple project types
- **Government contracts** — strict compliance needs
- Similar to MP Asuni AMP Salar (a previous government contractor Onsite worked with)

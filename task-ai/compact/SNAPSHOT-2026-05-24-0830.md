## Architecture state (current)

### 3-tier model routing (M-48 + M-49)
```
FAST_MODEL        openai/gpt-4o-mini       — fresh-chat onboarding chitchat (no anchor)
DEFAULT_MODEL     deepseek/deepseek-v4-pro — ALL anchored turns + all tool work
ESCALATION_MODEL  anthropic/claude-haiku-4-5 — when DeepSeek slips on UUID guard
COMPLEX_MODEL     anthropic/claude-sonnet-4-6 — last-resort, almost never
Embeddings        google/gemini-embedding-2-preview via OpenRouter (1536-dim)
Vision            google/gemini-2.5-flash → google/gemini-2.5-pro on validator fail
Voice STT         browser Web Speech API (native, free)
```

OpenRouter provider pinning for DeepSeek: `{ order: ['SiliconFlow', 'DeepSeek', 'Together', 'Nebius'], sort: 'price' }` — fixes the 6× cost variance bug.

### UUID guard stack (defense in depth)
1. **M-40 shape** — rejects position numbers ("3.1", "the-raft-work-ba-id")
2. **M-41 provenance** — rejects UUID-shaped fabrications never seen in conversation
3. **M-44 typed** — rejects right-shape, right-presence but wrong-type UUIDs (e.g. BA used as sub_activity)

### Project anchor (M-47)
- `task_ai_session_meta.project_anchor JSONB` stores `{project_id, project_name, company_id, company_name, anchored_at}`
- Server reads on every POST, injects into system prompt as hardcoded fact
- When `list_projects` returns exactly 1 match during onboarding → auto-anchor + auto-rename session
- If user mentions different project mid-chat → refuse + suggest "+ New chat"
